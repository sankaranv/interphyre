## Consistency with Hypothesis

**Largely consistent**, with one substantive deviation:

The hypothesis states "all 23 operational levels serve bundle entries generated by the
current valid-placement oracle." This is true for the 7 specifically-stale bundles (I1 +
I1b). However, 17 other bundles generated during `validation_repair` (before I4) lack
`oracle_commit` and their generation timestamp is not directly verifiable from the file.
Oracle correctness for those 17 is not in doubt (they use post-reform valid-placement
code), but the audit trail is incomplete.

The impossibility findings for `just_a_nudge` and `the_cradle` are consistent with the
hypothesis framing: the study certifies both as impossible and provides the prerequisite
evidence for level redesign.

   referenced basket_case, which was not in the I1 list. A new task (interphyre-mpi.1)
   was created to regenerate basket_case. The test for seed=6 (which shifted from variant=0
   to variant=4 in the new bundle) was updated to use seed=18.

2. **just_a_nudge bundle has 1 valid seed**: The regenerated bundle found seed=331 valid

   tested ramp-redirect and green_ball-push specifically; the basket-wall oracle finds one
   lucky geometry via random sampling at a high variant). The level remains flagged for
   redesign.

3. **35 tests pass (target: 37)**: The two-test gap is structural, not a failure — the
   O1/O2 tests are conditional on oracle deployment, and both were certified impossible.
